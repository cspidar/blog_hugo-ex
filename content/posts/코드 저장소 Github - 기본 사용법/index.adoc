== 왜 github?

* **git:** 소스코드를 관리하기 위한 버전 관리 시스템, 이력 관리와 협업과 관련된 다양한 기능을 제공
* **github:** git 을 인터넷에 업로드하는 서비스, 이외에도 유사 서비스 (gitlab 등) 이 있음

git은 브랜치, 모듈, 로그 등을 활용하여 개발환경에 훌륭하고 다양한 편의 기능을 제공한다. 하지만 복잡해보이는 수많은 개념 + 낯선 터미널 명령어는 이를 처음 접하는 사람들에게 장벽으로 다가온다고 생각한다. 따라서 git을 Google Drive 나 Naver Cloud 처럼 단순한 코드 + 파일의 원격 저장소의 개념으로 활용하는 방법만 소개하여 우선 github 서비스를 사용 가능하게 설명하고자 한다.

===== 저장소에 연결
방금 생성한 github의 저장소와 사용할 기기내 원하는 폴더와 연결

. 원하는 폴더를 Visual studio code 를 통해 열고 터미널 창을 켜거나 기타 터미널 프로그램을 통해 연다. Visual studio code 기본 설치 및 폴더 열기 방법은 <<빠른 시작>> 챕터 맨 처음 **(선택) IDE 설치 방법 확인: Visual studio code 항목에서 확인** 항목에서 확인.
+
. 원하는 폴더를 열었는지 확인 (아래에선 Github_test 폴더를 열었음)
+
image::images\index-849a9.png[]
+
. 다음 명령어 입력
+
.. git init: git 사용 준비
+
[source]
----
git init
----
+
image::images\index-20e4c.png[]
+
image::images\index-26103.png[]
+
[NOTE]
====
".git" 이라는 숨김 폴더가 프로젝트 폴더에 생성된다. 프로젝트의 업로드 관련 상태가 저장되어 있다고 생각하면 될듯 하다. 해당 폴더 (.git)를 지우면 프로젝트 폴더의 git 관련 상태를 초기화할 수 있다.
====
+
.. 만든 github 저장소의 "git remote add origin" 으로 시작하는 항목을 복사 후 터미널에 붙여넣어 입력 (원하는 폴더와 만든 github 저장소를 연결하는 명령어)
+
[source]
----
git remote add origin https://github.com/-----/-----.git
----
+
image::images\index-b9610.png[]

image::images\index-2b37c.png[]

[NOTE]
====
아무런 메세지도 나오지 않는것이 정상적으로 입력된 상태

===== 저장소에 올리기
.. git add .: 폴더내 전체 파일을 업로드 대상에 추가
+
image::images\index-257c0.png[]
+
.. git commit -m '커밋 메세지': 이번 업로드에 대한 설명 작성
+
image::images\index-268ff.png[]
+
.. git push origin main: 서버로 업로드 실행
+
image::images\index-7e212.png[]
+
[source]
----
git add .
git commit -m 'first commit'
git push origin main
----
+
[NOTE]
====
지금 (github를 웹하드처럼 사용하려는) 은 add, commit, push를 연속된 과정이라고 생각하자. 많은것을 빠뜨리고 이야기하지만, 심화 내용은 더 잘 설명되어있는 자료가 많으니 참고하면 좋을듯 하다.
====
+
[NOTE]
====
커밋 메세지는 github에 파일 및 코드가 업로드 되었을때를 구분하는 역할을 하며 아무 메세지나 입력해도 되지만, 보통은 이번에 올릴 코드나 파일에 대한 짧은 설명을 입력한다.
====

===== 다른 작업 환경에서 저장소 내용을 복사 (clone) 하기
. <<저장소에 연결>> 에서의 첫번째 과정과 같이, 원하는 폴더를 Visual studio code 를 통해 열고 터미널 창을 켜거나 기타 터미널 프로그램을 통해 연다.
+
. git clone "저장소 주소".git 을 입력하면 해당 폴더내 저장소 이름 폴더 안에 해당 저장소가 복사된다.
+
[source]
----
git clone https://github.com/-----/-----.git
----
